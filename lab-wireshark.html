LAB L01 · NETWORK SECURITY · BEGINNER

Wireshark + AI Traffic Analysis

Capture live packets from your lab network, analyze protocol distributions, apply Wireshark display filters, and use the AI Analyst to identify suspicious flows, decode attack signatures, and generate network-based IOCs.

⏱ 45 min
🖥️ Kali Linux
🔧 Wireshark · tcpdump · tshark
🤖 AI Traffic Analysis
📡 Lab Scenario

Your SOC has flagged unusual outbound traffic from a workstation. You've been given a PCAP file and asked to determine: what protocols are in use, are there any suspicious connections, and is there evidence of data exfiltration or C2 communication?

In this lab you'll capture live traffic between your Kali VM and Metasploitable, analyze it in Wireshark, and use the AI Analyst to interpret your findings.

0
Phase 0 — Setup & Capture
Get Wireshark running and capture your first PCAP
Verify lab network connectivity
Confirm Kali can reach Metasploitable before capturing
Ensure both VMs are on the Host-Only network. From Kali:
ping -c 3 192.168.56.101 # Should reach Metasploitable
ip addr show # Find your interface name (eth0/eth1)
nmap -sn 192.168.56.0/24 # Discover all hosts on the network
Start a background packet capture with tcpdump
Capture all traffic to a PCAP file while you generate activity
# Start capture in background (replace eth0 with your interface)
sudo tcpdump -i eth0 -w /tmp/lab01_capture.pcap &
CAPTURE_PID=$!

# Generate diverse traffic to Metasploitable
ping -c 5 192.168.56.101
nmap -sV -p 22,80,21,23,3306 192.168.56.101
curl -s http://192.168.56.101 > /dev/null
ftp 192.168.56.101 <<< "quit"

# Stop capture
sudo kill $CAPTURE_PID
ls -lh /tmp/lab01_capture.pcap
Open the PCAP in Wireshark
Load and explore the capture file
wireshark /tmp/lab01_capture.pcap &
In Wireshark: go to Statistics → Protocol Hierarchy to see protocol distribution. Screenshot this — you'll share it with the AI Analyst.

Phase 1 — Display Filters & Analysis
Use Wireshark filters to isolate traffic types and find anomalies
Apply essential display filters
Isolate protocols and connections of interest
Enter these filters in the Wireshark display filter bar one at a time. Note what you see for each.
# Paste each into Wireshark's filter bar:
tcp.flags.syn == 1 && tcp.flags.ack == 0 # SYN packets = new connections
tcp.flags.rst == 1 # RST = rejected connections
ftp # FTP traffic (plaintext creds!)
http # HTTP requests/responses
icmp # Ping traffic
ip.addr == 192.168.56.101 # All traffic to/from target
tcp.port == 23 # Telnet — plaintext protocol
Analyze conversations and endpoints
Statistics menus reveal the big picture of who's talking to whom
Open these Statistics menus and record your findings:
# In Wireshark menus:
Statistics → Conversations # All TCP/UDP conversations, data volume
Statistics → Endpoints # All IPs and ports seen
Statistics → Protocol Hierarchy # Breakdown by protocol %
Analyze → Expert Information # Errors, warnings, anomalies flagged
🤖 AI Analyst

Paste the conversation summary (IPs, ports, data volumes) from the Conversations window into the AI chat. The AI will identify suspicious connections and explain what each protocol usage pattern suggests.

Follow TCP streams
Reconstruct full sessions to read plaintext credentials and commands
Right-click any TCP packet → Follow → TCP Stream. Do this for FTP, HTTP, and Telnet packets. You'll see the full plaintext conversation including any credentials.
💡
FTP and Telnet transmit credentials in cleartext. This is why they're banned in modern environments. You should be able to see the username and password in the stream.
Use tshark for command-line analysis
tshark enables scriptable, automated packet analysis
# Extract all unique IP conversations
tshark -r /tmp/lab01_capture.pcap -q -z conv,tcp

# Find all HTTP requests
tshark -r /tmp/lab01_capture.pcap -Y "http.request" -T fields -e http.host -e http.request.uri

# Extract DNS queries
tshark -r /tmp/lab01_capture.pcap -Y "dns.qry.name" -T fields -e dns.qry.name | sort -u

# Find large data transfers (potential exfiltration)
tshark -r /tmp/lab01_capture.pcap -q -z io,stat,0 -z "io,stat,0,ip.addr==192.168.56.101"

Phase 2 — Nmap Scan Detection
Capture and identify what a port scan looks like in Wireshark
Capture an Nmap scan in real time
See exactly what a port scan looks like at the packet level
# Terminal 1: Start capture
sudo tcpdump -i eth0 -w /tmp/nmap_scan.pcap host 192.168.56.101

# Terminal 2: Run scans
nmap -sS 192.168.56.101 # SYN stealth scan
nmap -sU -p 53,161 192.168.56.101 # UDP scan
nmap -sV 192.168.56.101 # Version detection

# Terminal 1: Stop capture (Ctrl+C)
wireshark /tmp/nmap_scan.pcap &
💡
Filter: tcp.flags.syn==1 && tcp.flags.ack==0 — you'll see hundreds of SYN packets to sequential ports. This is the signature of a SYN scan.
Identify scan patterns with AI
Use the AI Analyst to characterize the scan type and write detection signatures
🤖 AI Analyst — Scan Detection

Export 20-30 rows from the Wireshark packet list (File → Export Packet Dissections → As CSV) and paste into the AI. It will identify the scan type, explain the TCP flag patterns, and write a Snort/Suricata signature to detect this scan.

Phase 3 — AI IOC Extraction & Report
Synthesize all findings into network-based IOCs and a detection summary
Export objects and identify data exfiltration indicators
Recover files transferred over HTTP, FTP, or SMB
In Wireshark: File → Export Objects → HTTP (or FTP-DATA). This recovers any files transferred during your capture session. Also useful for malware C2 analysis.
# CLI equivalent using tshark
tshark -r /tmp/lab01_capture.pcap --export-objects http,/tmp/http_objects/
Generate network IOCs with AI
Convert your findings into structured threat intel
